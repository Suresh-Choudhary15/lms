# Project Overview

This project is a modern self-paced Learning Management System (LMS) inspired by platforms such as Udemy, Coursera, and Skillshare.

The assignment is intentionally time-boxed to **24 hours** and explicitly encourages an **AI-first development workflow**.

The objective is **not** to recreate every enterprise feature.

The objective is to demonstrate:

- Product thinking
- Engineering judgment
- Software architecture
- UI/UX quality
- Clean code
- AI-assisted development
- Ability to prioritize correctly

This project should feel like **Version 1.0 of a real startup product**, not a hackathon prototype.

---

# Assignment Summary

Create a self-paced LMS that allows users to:

- Browse courses
- View course details
- Enroll in a course
- Watch video lessons
- Track learning progress
- Navigate lessons intuitively

Use 2–3 embedded YouTube videos as sample content.

Deploy the application and provide a live URL.

---

# Primary Goal

Deliver the **highest-quality user experience** possible within the available time.

Quality is more important than quantity.

Every completed feature is expected to feel production-ready.

---

# Product Philosophy

This LMS should communicate:

- Simplicity
- Professionalism
- Trust
- Accessibility
- Clarity
- Consistency

The interface should never overwhelm the user.

Every screen should have one obvious purpose.

Navigation should require almost no thinking.

---

# Product Principles

## 1. Clarity over Complexity

If a feature makes the interface more confusing,
it should not be built.

---

## 2. Quality over Quantity

Five polished features are better than twenty unfinished ones.

---

## 3. Consistency over Creativity

Maintain a consistent experience across the application.

Spacing

Typography

Buttons

Cards

Colors

Animations

Interactions

Everything should feel like one product.

---

## 4. User First

Every decision must answer:

"Does this improve the student's learning experience?"

If not,

do not build it.

---

## 5. AI is a Team Member

AI is not merely generating code.

AI acts as:

- Software Architect
- Product Designer
- Senior Engineer
- Reviewer
- QA Engineer
- Documentation Writer

Every implementation should be planned before coding.

# Non-Negotiable Rules

Never sacrifice quality for feature count.

Never duplicate components.

Never overengineer.

Never introduce unnecessary abstractions.

Never leave placeholder UI.

Never leave TODO comments.

Never commit broken functionality.

Never build features outside the roadmap without a clear reason.
