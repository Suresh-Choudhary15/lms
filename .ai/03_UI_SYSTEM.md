# UI_SYSTEM.md

> "People don't remember every feature.
> They remember how effortless the product felt."

---

# Purpose

This document defines the visual language, interaction patterns, and user experience principles for the Learning Management System.

Every page should feel like it belongs to the same product.

The UI should communicate:

- Professionalism
- Simplicity
- Trust
- Focus
- Modern craftsmanship

This is not a dashboard full of widgets.

It is a learning experience.

---

# Design Inspiration

The visual quality should be inspired by:

- Udemy
- Linear
- Vercel
- Notion
- Stripe Dashboard
- Raycast
- GitHub

Not by copying them.

Instead, capture their design philosophy:

- Clean layouts
- Generous spacing
- Excellent typography
- Strong hierarchy
- Subtle animations
- Minimal distractions

---

# Visual Principles

## 1. Learning Comes First

The interface should never compete with lesson content.

Videos.

Lesson text.

Navigation.

Progress.

Everything else is secondary.

---

## 2. Less Noise

Avoid unnecessary gradients.

Avoid large illustrations.

Avoid decorative elements that do not improve usability.

White space is a design element.

---

## 3. Consistency

Every screen should feel familiar.

Buttons.

Cards.

Typography.

Spacing.

Animations.

Navigation.

Everything should follow one system.

---

# Layout System

Maximum content width:

1280px

Content container:

Centered

Consistent horizontal padding:

Desktop

Tablet

Mobile

Never let content stretch edge-to-edge.

---

# Grid System

Desktop:

12-column mindset

Tablet:

6-column mindset

Mobile:

Single-column flow

Cards should naturally reflow.

Never horizontally scroll.

---

# Spacing Scale

Use a consistent spacing rhythm.

Tiny

Small

Medium

Large

Extra Large

Never use arbitrary spacing values.

Spacing should feel predictable.

---

# Border Radius

Soft corners.

Modern.

Avoid both extremes:

Too sharp.

Too rounded.

Consistency matters more than the exact radius.

---

# Shadows

Very subtle.

Only to establish depth.

Cards should appear elevated—not floating.

Avoid heavy drop shadows.

---

# Color Philosophy

Neutral-first.

Accent-second.

The content should be more colorful than the interface.

Avoid rainbow dashboards.

Use accent colors intentionally.

Color should communicate state—not decoration.

---

# Typography

Readable before beautiful.

Use clear hierarchy.

Page Title

↓

Section Heading

↓

Card Title

↓

Body

↓

Caption

Never rely solely on color for hierarchy.

Use spacing and font weight.

---

# Icon Usage

Icons support recognition.

They never replace labels.

Prefer Lucide icons.

Every icon should have purpose.

Avoid icon overload.

---

# Buttons

Primary

One per screen.

Secondary

Supporting actions.

Ghost

Low emphasis.

Destructive

Only for irreversible actions.

Never place multiple competing primary buttons together.

---

# Forms

Simple.

Short.

Clear labels.

Visible validation.

Helpful error messages.

Logical tab order.

Never hide required information.

---

# Cards

Cards are the primary content container.

Every card should contain:

Clear title

Supporting information

Primary action

Consistent padding

Consistent spacing

Hover feedback

Avoid overcrowding.

---

# Navigation

Navigation should answer:

Where am I?

Where can I go?

How do I go back?

Users should never feel lost.

---

# Dashboard Experience

When users open the dashboard they should immediately understand:

Continue Learning

↓

My Courses

↓

Progress

↓

Recommended Courses

↓

Recent Activity

Most important information first.

---

# Course Listing

Each course card should quickly communicate:

Thumbnail

Title

Instructor

Duration

Difficulty

Progress (if enrolled)

Primary action

Cards should scan easily.

---

# Course Detail Page

Information hierarchy:

Hero

↓

Course Overview

↓

Learning Outcomes

↓

Curriculum

↓

Instructor

↓

Call to Action

Avoid overwhelming users with information.

---

# Lesson Experience

This is the heart of the product.

Prioritize:

Video

↓

Lesson Navigation

↓

Progress

↓

Lesson Notes (placeholder)

↓

Previous / Next

The learning experience should never feel cluttered.

---

# Sidebar

The sidebar should remain predictable.

Sections:

Course Progress

↓

Chapters

↓

Lessons

Current lesson should always be obvious.

Completed lessons should be visually distinct.

---

# Progress Visualization

Progress should always be visible.

Use:

Progress bars

Completion percentages

Completed lesson indicators

Users should feel momentum.

---

# Empty States

Every empty state should answer:

Why is this empty?

What should I do next?

Never show a blank page.

---

# Loading States

Prefer skeletons.

Maintain layout stability.

Avoid flashing content.

---

# Error States

Friendly.

Actionable.

Never expose technical details.

Always offer recovery.

---

# Animations

Animations should:

Guide attention.

Confirm actions.

Smooth transitions.

Never become entertainment.

Subtle > Fancy.

---

# Microinteractions

Hover

Focus

Click

Completion

Progress updates

Navigation

These small moments create perceived quality.

---

# Responsive Design

Desktop first.

Then optimize:

Tablet

Mobile

Every screen should feel intentionally designed.

Not merely shrunk.

---

# Accessibility

Every interactive element must support:

Keyboard navigation

Visible focus

Screen readers

Proper labels

Adequate contrast

Accessibility improves usability for everyone.

---

# Dark Mode

Support both light and dark themes.

Colors should be designed—not inverted.

Maintain contrast.

Maintain hierarchy.

Maintain readability.

---

# Visual Quality Checklist

Before considering any screen complete:

□ Consistent spacing

□ Clear hierarchy

□ Predictable navigation

□ Responsive

□ Accessible

□ Smooth interactions

□ Appropriate empty state

□ Appropriate loading state

□ Appropriate error state

□ Consistent component styling

□ No visual clutter

□ Excellent readability

---

# Product Feel

If someone uses this LMS for five minutes, they should think:

"This feels polished."

Not:

"This looks AI-generated."

Every design decision should reinforce trust, clarity, and focus.

Build an interface that disappears behind the learning experience.

That is the highest compliment a learning platform can receive.
